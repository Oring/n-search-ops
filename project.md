# 본스탬프 내 네이버 검색 시스템 구축 기획안

---

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| **목적** | 본스탬프 앱 사용자가 지정된 주차에 특정 키워드로 네이버 검색을 하도록 유도 |
| **대상** | 수동검색자 테스터 (초기 약 100~120명 예상) |
| **주요 기능** | 검색 자동화 버튼, 그룹별 키워드 할당 시스템, 액션 로그 기록, 어드민 통계 화면 |

---

## 1-1. 운영 전제

- 본 시스템은 **기존 회원/예약/등록/출석 백엔드와 완전히 분리된 별도 환경**에서 운영
- 기존 서비스와 직접 연동하지 않고, 관리자가 필요한 테스터만 별도로 등록하여 관리
- 테스터 식별 정보는 최소한으로 저장
  - 예) 회원번호, 이름 또는 휴대폰 뒷자리, 소속 그룹, 활성 여부
- 관리자는 등록된 테스터에게 그룹별/주차별 키워드를 할당하고, 앱은 이 분리된 시스템에만 요청

### 앱 API 구성(초안)

- **조회 API**: 앱 사용자의 회원번호를 받아, 오늘 검색 대상인지 확인
  - 대상이 아니거나 이미 오늘 검색을 완료한 경우: 빈 응답
  - 대상인 경우: 오늘 수행할 검색 키워드 반환
- **기록 API**: 앱이 검색 시작(버튼 클릭) 사실을 서버에 전송
  - 서버는 회원번호, 그룹, 키워드, 시각 등 필요한 정보를 기록
  - 기록된 데이터는 관리자 로그/통계 화면에서 조회

### 기술 방향(결정)

- **Backend**: `FastAPI`
- **Admin Frontend**: `Jinja2 + HTMX + Alpine.js`
- **배포**: `Render`

## 2. 테스터 그룹 및 키워드 할당 시스템 `Backend/Logic`

### ① 그룹 관리 및 테스터 할당

- 테스터들을 A, B, C, D 등 여러 그룹으로 생성하고 관리할 수 있는 구조
- 관리자는 개별 테스터를 특정 그룹에 자유롭게 할당/변경할 수 있어야 함

### ② 주차별 테스크(키워드) 매핑

- **그룹별 키워드 할당**: 특정 주차(Week)에 각 그룹이 작업할 키워드를 관리자가 미리 세팅
  - 예) 1주차 - A그룹: `"키워드1"`, B그룹: `"키워드2"` 할당
- **자동 키워드 연동**: 테스터가 앱에서 버튼을 클릭할 때, 본인이 속한 그룹에 해당 주차에 할당된 키워드가 자동으로 검색어(query)로 입력되어 네이버로 이동

---

## 3. 데이터 기록 및 로깅 `Backend/Log`

### ① 개별 테스터 활동 기록

- **기록 시점**: 테스터가 앱 내에서 검색 버튼을 클릭하는 순간
- **필수 수집 항목**
  - 테스터 식별 정보 (ID / 이름 / 소속 그룹)
  - 클릭 일시 (Timestamp)
  - 당시 할당되어 클릭된 키워드명

### ② 테스터 운영 계획

- **권장 횟수**: 인당 월 2회 실시 권장
- **운영 로직**: 그룹별/주차별 로테이션을 통해 작업량이 분산되도록 관리

---

## 4. 관리자 페이지 `Admin Side`

### ① 실시간 작업 현황 대시보드

- **날짜별 통계**: 키워드당 일자별 총 클릭 수(작업량) 그래프 및 표
- **키워드별 통계**: 각 키워드별 누적 작업량 집계
- **테스터별 통계**: 각 테스터 인원별 참여도 및 작업 현황 (추후 보상체계 등과도 연계)

### ② 테스터 상세 로그 조회 및 관리

- **필터 기능**: 날짜별, 그룹별, 키워드별, 테스터별 상세 검색 결과 필터링
- **데이터 추출**: 통계 및 로그 데이터의 엑셀(CSV) 다운로드 기능
- **할당 관리**: 주차별 키워드-그룹 매핑 설정 화면
